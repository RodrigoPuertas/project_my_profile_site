Forecasting & Séries Temporais

Como prever demanda e comportamento usando séries temporais: ARIMA, Prophet e modelos de ML. Aplicado diretamente ao S&OP.

O que é Forecasting

Forecasting é a disciplina de prever valores futuros com base em dados históricos. No contexto de S&OP (Sales & Operations Planning), isso significa prever demanda, volume de pacotes, capacidade necessária e recursos logísticos com antecedência suficiente para que decisões estratégicas sejam tomadas a tempo.

A diferença entre forecast e chute é simples: forecast tem método, métrica e rastreabilidade. Você sabe como chegou no número, pode medir o erro, e pode reproduzir o resultado.

Definição: Forecasting é o processo de estimar valores futuros de uma variável com base em padrões identificados nos dados históricos, usando métodos estatísticos ou de Machine Learning.

Horizontes de previsão

  • Curto prazo (1-7 dias) — operação diária, alocação de equipes, capacidade de sorting centers
  • Médio prazo (1-3 meses) — planejamento de frota, contratação, expansão de hubs
  • Longo prazo (6-12 meses) — decisões de investimento, abertura de novas operações, FP&A

Componentes de uma Série Temporal

Toda série temporal pode ser decomposta em componentes fundamentais. Entendê-los é o primeiro passo para escolher o modelo certo.

  • Tendência (Trend) — direção geral de longo prazo (crescimento ou queda)
  • Sazonalidade (Seasonality) — padrões que se repetem em períodos fixos (dia da semana, mês, feriados)
  • Resíduo (Residual) — a parte que não é explicada por tendência nem sazonalidade (ruído)

Métodos Estatísticos Clássicos

ARIMA (AutoRegressive Integrated Moving Average)

O ARIMA é o modelo clássico para séries temporais. Combina três componentes definidos pelos parâmetros (p, d, q):

  • p (AR) — número de termos autorregressivos (dependência dos valores passados)
  • d (I) — grau de diferenciação para tornar a série estacionária
  • q (MA) — número de termos de média móvel (dependência dos erros passados)

Para séries com sazonalidade, usa-se o SARIMA que adiciona parâmetros sazonais (P, D, Q, m).

Exponential Smoothing (Holt-Winters)

Método que atribui pesos exponencialmente decrescentes aos dados passados. O Holt-Winters lida com tendência e sazonalidade ao mesmo tempo. É simples, rápido e surpreendentemente eficaz para muitos cenários.

Prophet: forecasting acessível

Desenvolvido pelo Facebook (Meta), o Prophet é um modelo aditivo que decompoe a série em tendência + sazonalidade + feriados + ruído. Seu grande diferencial é a facilidade de uso e a capacidade de lidar com dados faltantes e outliers. 


from prophet import Prophet
import pandas as pd

# Dados no formato ds (data) + y (valor)
df = pd.read_csv("demanda_diaria.csv")

# Instanciar e treinar
model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False,
    changepoint_prior_scale=0.05
)
model.fit(df)

# Prever 30 dias à frente
future = model.make_future_dataframe(periods=30)
forecast = model.predict(future)

# Visualizar
model.plot(forecast)
model.plot_components(forecast)

Modelos de ML para Séries Temporais

Além dos métodos estatísticos, modelos de ML como XGBoost e LightGBM podem ser adaptados para forecasting transformando o problema em um problema de regressão com features temporais.

Métricas de avaliação

Um modelo de forecast só é tão bom quanto a métrica usada para avaliá-lo. As principais:

  • MAE (Mean Absolute Error) — erro médio absoluto. Fácil de interpretar, mesma unidade dos dados.
  • RMSE (Root Mean Squared Error) — penaliza erros grandes. Útil quando outliers são críticos.
  • MAPE (Mean Absolute Percentage Error) — erro percentual. Bom para comparar entre séries de escalas diferentes.
  • SMAPE (Symmetric MAPE) — versão simétrica do MAPE, mais robusta quando valores reais são próximos de zero.

Forecasting em produção: S&OP na prática

Na área de S&OP da Shopee Express, o forecast não é um exercício acadêmico — é o número oficial que direciona decisões de capacidade, contratação e investimento. Isso exige uma camada extra de engenharia além do modelo.

Cenários de produção vs. validação

O modelo de produção gera o número oficial. Paralelamente, mantemos cenários de validação com configurações alternativas (features diferentes, janelas de treino distintas, hiperparâmetros otimizados). Comparamos os dois diariamente para detectar degradação.

Âncoras e janelas de validade

Uma âncora é o ponto de referência temporal a partir do qual o forecast é gerado. A janela de validade define até quando aquele forecast é válido. Quando a janela expira, o modelo é re-treinado com dados mais recentes.

Governança de forecast: em produção, cada número precisa de: versão do modelo, data da âncora, janela de treino, features utilizadas, e métricas de validação. Sem rastreabilidade, não há confiança.

Versionamento

Cada rodada de forecast é versionada com metadados completos: versão do código, versão dos dados de treino, hiperparâmetros e métricas. Isso permite auditoria — quando alguém pergunta "por que o forecast errou naquela semana?", você pode rastrear até a causa raiz.

Código prático: forecast com statsmodels


import pandas as pd
from statsmodels.tsa.holtwinters import ExponentialSmoothing

# Carregar dados
df = pd.read_csv("demanda.csv", parse_dates=["date"], index_col="date")

# Treino / teste
train = df[:-30]
test = df[-30:]

# Holt-Winters com sazonalidade multiplicativa
model = ExponentialSmoothing(
    train["volume"],
    trend="add",
    seasonal="mul",
    seasonal_periods=7
).fit()

# Prever 30 dias
forecast = model.forecast(30)

# Avaliar
from sklearn.metrics import mean_absolute_error
mae = mean_absolute_error(test["volume"], forecast)
print(f"MAE: {mae:.0f}")
